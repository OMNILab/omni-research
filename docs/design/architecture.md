# Architecture and Repository Structure

```
omni-research/
├── docs/                  # Project documentation (this skill set only)
│   ├── design/            # Design documents (topic-centric)
│   └── brainstorming/     # Brainstorming session outputs
├── skills/                # Agentic skills source
```

## Directory Purposes

### `raw/` — Raw Material Layer
- **Purpose**: Store unprocessed materials from various sources
- **Classification**: Tier-based (paper, web, github, model, dataset)
- **Format**: PDF, markdown, code, checkpoint files
- **Indexing**: Automatic metadata extraction on collection

### `docs/plans/` — Formal Artifacts
- **Purpose**: Store structured, traceable research artifacts
- **Naming**: `{type}-{id}-v{version}.md`
- **Status**: draft → reviewed → published → archived
- **Dependencies**: Explicit tracking via metadata frontmatter

### `docs/index/` — Machine-Readable Indexes
- **Purpose**: Provide structured data for tooling integration
- **Format**: JSON (machine-readable) + markdown (human-readable)
- **Content**: Artifact registry, version history, traceability matrix
- **Updates**: Automatic on artifact creation/update

### `docs/archive/` — Historical Records
- **Purpose**: Preserve superseded artifact versions
- **Trigger**: Automatic on reconciliation, manual on user request
- **Structure**: Timestamped snapshots with index
- **Read-only**: Archived artifacts cannot be modified

### `src/` — Generated Code
- **Purpose**: Prototype implementations and evaluation code
- **Generated by**: `omr-evaluation`
- **Structure**: Prototype, evaluation, tools
- **Versioning**: Follows git versioning, not artifact metadata

### `wiki/` — Living Knowledge Base
- **Purpose**: Quick reference, continuously updated
- **Generated by**: `omr-wiki`
- **Links**: Back-references to source documents
- **Format**: Interlinked concept pages
